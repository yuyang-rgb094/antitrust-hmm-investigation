# 反垄断调查 HMM 概率推演框架

## 用途

当标的公司（或行业竞争对手）遭遇反垄断/价格串通调查时，使用本框架进行系统性分析：
1. 事件全景与历史检索（涉案方、调查机构、时间线）
2. 三公司竞争格局分析（市场份额、客户依赖、财务对比）
3. 股价行为分析（事件冲击、量价关系、相对强弱）
4. **隐马尔可夫模型（HMM）概率推演**（核心）
5. 投资决策矩阵（各情景下的定价区间与应对策略）

## 前置条件

- Wind MCP 工具可用（获取行情、基本面、客户/营收结构数据）
- 联网搜索工具可用（检索新闻、调查进展、诉讼信息）
- 标的公司基本信息（代码、业务结构）
- Python 环境（运行 `hmm_forward.py`，依赖 numpy）

## 工具文件

- `hmm_forward.py` — HMM 前向算法 Python 实现
- `SKILL.md` — 本文件

## 工作流程

### 第一步：事件全景与历史检索

使用联网搜索 + Wind MCP 检索以下信息：

1. **调查时间线**
   - 首次披露时间与方式（招股书、公告、新闻）
   - 各阶段调查进展（突击搜查→立案→移交审判→判决）
   - 涉及的国家/地区（美国DOJ、韩国KCC/FTC、欧盟等）
   - 涉及的法律（谢尔曼法、韩国《垄断规制与公平交易法》等）

2. **涉案公司基本面**
   - 各公司主营业务结构
   - 核心客户及营收占比（特别是客户集中度）
   - 市场份额数据
   - 近期财务数据

3. **类似案例参考**
   - 行业历史反垄断案例的处理方式和结果
   - 罚款/和解金额范围
   - 对业务的实际影响

### 第二步：三公司竞争格局分析

使用 Wind MCP 获取以下数据：

| 维度 | 数据项 | 数据源 |
|:----:|:-------|:-------|
| 份额 | 全球市场份额、市场集中度CR3/CR5 | Wind/行业报告 |
| 客户 | 核心客户营收占比、客户行业分布 | Wind年报/招股书 |
| 财务 | 营收、净利润、毛利率、研发费用 | Wind |
| 估值 | PE(TTM)、PE(预测)、PS | Wind |
| 行情 | 日K线（事件前后）、成交量、换手率 | Wind |

**关键分析点：**
- 客户集中度越高，调查对议价能力的威胁越大
- 业务多元化程度越低，调查对基本面的冲击越大
- 历史"前科"影响调查结果预期

### 第三步：股价行为分析

1. **事件冲击度量**
   - 事件日跌幅/涨跌幅
   - 事件后1/3/5/10个交易日累计收益
   - 与大盘/板块指数的相对强弱（α）

2. **量价特征**
   - 事件日成交量/换手率（历史分位数）
   - 盘中走势（V型/倒V型/持续单边）
   - 尾盘信号（反弹/继续走弱）

3. **跨公司联动**
   - 三家公司的股价相关性
   - 哪家跌幅最大（分析原因：估值/份额/客户结构）
   - 被错杀可能性判断

### 第四步：隐马尔可夫模型（HMM）概率推演 ★核心

#### 4.1 定义隐含状态 S

根据调查进展程度定义 4 个隐含状态：

| 状态 | 描述 | 特征 |
|:----:|:-----|:------|
| S1 | 无果结案 | 调查最终证据不足，以轻微违规或低额罚款结案 |
| S2 | 和解罚款 | 认定部分违规，支付和解金+整改 |
| S3 | 实质性处罚 | 严重违规认定，高额罚款+客户转单+价格重谈 |
| S4 | 极端连锁反应 | 全球多国平行调查，业务结构被迫重塑 |

#### 4.2 初始状态概率分布 π

基于已知信息（招股书披露、既往调查、行业惯例）设定初始概率：

```
π = [P(S1), P(S2), P(S3), P(S4)]
```

**设定原则：**
- 如果调查已有初步结论/指引 → 调整对应概率
- 如果首次调查、无前科 → S2（和解）概率最高
- 如果已有平行调查 → S3/S4 概率上调

#### 4.3 状态转移矩阵 A

```
        S1     S2     S3     S4
S1  [  0.95,  0.04,  0.01,  0.00 ]
S2  [  0.05,  0.85,  0.08,  0.02 ]
S3  [  0.00,  0.10,  0.80,  0.10 ]
S4  [  0.00,  0.00,  0.15,  0.85 ]
```

**说明：**
- 对角线高值表示状态有惯性（调查进展有路径依赖）
- S1→S4 概率极低（无果结案跳变到极端事件几乎不可能）
- S4→S4 保持高概率（极端事件一旦触发会持续发酵）

#### 4.4 观测矩阵 B

定义 7 个可观测事件，以及各状态下的观测概率：

| 观测 | 描述 | 最可能状态 |
|:----:|:-----|:----------:|
| O1 | 韩国检方无新进展，公司回应"配合调查" | S1 |
| O2 | 检方公布初步调查结论 | S2/S3 |
| O3 | 检方移交审判/起诉高管 | S3/S4 |
| O4 | 公司公告大额罚款计提 | S3 |
| O5 | 客户启动替代供应商验证 | S3/S4 |
| O6 | 美国DOJ宣布调查结果/罚款 | S3/S4 |
| O7 | 三星/海力士/Micron发布降价/赔偿公告 | S2/S3 |

**观测矩阵 B（示例）：**
```
        O1    O2    O3    O4    O5    O6    O7
S1  [  0.50, 0.20, 0.02, 0.01, 0.01, 0.10, 0.16 ]
S2  [  0.15, 0.35, 0.05, 0.05, 0.05, 0.10, 0.25 ]
S3  [  0.02, 0.15, 0.20, 0.15, 0.15, 0.10, 0.23 ]
S4  [  0.01, 0.05, 0.15, 0.10, 0.20, 0.20, 0.29 ]
```

#### 4.5 使用 Python 工具执行前向算法

```bash
# 使用默认参数（澜起案例：3步递推）
python hmm_forward.py

# 自定义初始概率（反映事件最新进展）
python hmm_forward.py --pi 0.1 0.3 0.4 0.2 --obs O1 O2 O3 --verbose

# 自定义观测序列（实际观测到的事件）
python hmm_forward.py --obs O1 O2 O3 O4 O5

# JSON 输出（便于程序化调用）
python hmm_forward.py --pi 0.05 0.25 0.45 0.25 --obs O1 O2 O3 O4 --json
```

#### 4.6 结果解读

| 概率区间 | 评级 | 应对策略 |
|:--------:|:-----|:---------|
| S1 > 40% | 偏乐观 | 调查大概率无果结案，被错杀标的可逢低布局 |
| S2 > 40% | 中性 | 和解罚款是大概率，需要评估罚款对财务的影响幅度 |
| S3 > 20% | 警惕 | 实质性处罚风险不可忽视，需要调整仓位 |
| S4 > 5% | 高风险 | 极端连锁反应可能性存在，需大幅减仓 |

### 第五步：投资决策矩阵

#### 5.1 各情景定价区间

基于历史估值中枢和事件冲击幅度，估算各情景下的估值区间：

| 情景 | 估值区间 | 假设条件 |
|:----:|:---------|:---------|
| S1（无果结案） | 事件前价格 × [0.85-1.00] | 修复行情，但调查阴影需时间消散 |
| S2（和解罚款） | 事件前价格 × [0.70-0.90] | 罚款一次性影响，业务基本盘不变 |
| S3（实质性处罚） | 事件前价格 × [0.50-0.70] | 客户转单+罚款双重打击 |
| S4（极端连锁） | 事件前价格 × [0.30-0.50] | 业务结构重塑，基本面恶化 |

#### 5.2 关键观测变量监控清单

| 观测 | 触发时更新HMM | 触发时建议操作 |
|:----:|:-------------|:---------------|
| O1 | 重新运行HMM，S1概率上升 | 持有/加仓 |
| O2 | 重新运行HMM，S2/S3概率重新分配 | 减仓至中性仓位 |
| O3 | 立即重新运行HMM，S3/S4概率上升 | 大幅减仓 |
| O4 | 重新运行HMM，S3概率确认 | 减仓 |
| O5 | 立即重新运行HMM，S3/S4概率大幅上升 | 大幅减仓或清仓 |
| O6 | 重新运行HMM | 视结果决定 |
| O7 | 重新运行HMM | 视结果决定 |

#### 5.3 仓位管理

```
仓位比例 = 基准仓位 × (1 - S3概率 - 2×S4概率)
```

其中基准仓位为无事件影响时的正常仓位。

### 第六步：跨公司差异分析

分析三家公司面临同一调查的不同影响程度：

| 分析维度 | 关键问题 |
|:---------|:---------|
| 客户集中度 | 哪家对涉案客户的依赖最大？ |
| 业务多元化 | 哪家业务最单一，风险最集中？ |
| 历史前科 | 哪家此前已有反垄断经验，有应对机制？ |
| 市场反应 | 哪家跌幅最大/最小，是否合理？ |
| 安全边际 | 哪家在S1/S2情景下当前估值已具备安全边际？ |

### 第七步：结论输出格式

```markdown
## 结论总结

### HMM概率分布
- S1（无果结案）：XX%
- S2（和解罚款）：XX%
- S3（实质性处罚）：XX%
- S4（极端连锁）：XX%

### 最可能路径
[简述1-2个最可能的调查发展方向]

### 各公司推荐
| 公司 | 当前价 | S1估值 | S2估值 | 当前策略 |
|:----:|:------:|:------:|:------:|:---------|

### 关键风险
1. 观测变量O3/O5触发将改变概率分布
2. 时间跨度风险（调查可能持续18-24个月）
3. 诉讼/集体诉讼的附加赔偿风险

### 数据来源
- Wind MCP：行情、财务、估值数据
- 联网搜索：新闻、调查进展
- 公司公告/招股书

> 数据口径标注：[FACT]来自Wind/公告 [OPINION]模型假设 [INFERENCE]基于HMM推算
```

## 输出规范

1. 所有金融数据标注来源（Wind MCP、公司公告、联网搜索）
2. 区分 FACT / OPINION / INFERENCE
3. 标注不确定性程度（高/中/低）
4. 输出前做"数据溯源检查"：每个数字的来源是什么？口径是什么？
5. HMM参数（初始概率π、转移矩阵A、观测矩阵B）必须明确展示
6. 前向算法递推过程必须完整展示（至少3步）

## 数据纪律

- 所有金融数据（行情、财务、估值）的**默认路径 = Wind MCP**
- 联网搜索只用于定性信息（新闻、公告解读、行业趋势）
- 每次引用数据标注来源和日期
- 市盈率口径严格区分：PE(LYR) / PE(TTM) / PE(预测) / PE(远期)
